# Documentation Standard

Status: Draft
Owner: Konstantin / Izumi IT
Purpose: Keep IAOS documentation concise, useful, and source-of-truth safe.

## Format

Documentation is Markdown-first.

Prefer short practical documents over long abstract explanations.

## Required header

Every IAOS document should include status, owner, and purpose.

When useful, also include scope and update rule.

## Purpose rule

Every document must help make decisions, build products, sell, support clients, reduce repeated work, reduce risk, or reduce decision fatigue.

If it does not help, do not create it yet.

## Source-of-truth rule

Avoid duplicate source of truth.

If another file or system owns the information, link to it or summarize only the operating rule.

## Generic knowledge rule

Do not store generic internet knowledge in IAOS unless it becomes an Izumi IT standard, decision, or workflow.

## Update rule

Update documents when real practice changes.

Do not update documents just to make them look complete.
