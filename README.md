# IAOS Foundation v0.1 Draft

Status: Draft
Owner: Konstantin / Izumi IT
Purpose: Provide the practical foundation for how Izumi IT builds, decides, documents, and improves.

## What IAOS is

IAOS is the company operating system for Izumi IT.

It is a Markdown-first source of truth for product and engineering workflows, AI-assisted development practices, decision records, documentation standards, reusable prompts, and company knowledge that should survive specific tools and AI providers.

IAOS also preserves the original mission from `name.txt`: create a long-living, vendor-independent system for managing Izumi IT knowledge, processes, and engineering standards so the company can build high-quality products for the Japanese B2B market.

## Why IAOS exists

Izumi IT is currently founder-led and uses AI-assisted development to move quickly. IAOS exists to keep that speed practical, reviewable, and repeatable without adding bureaucracy.

IAOS should help Izumi IT build real products now, reduce repeated decisions, reduce avoidable risk, keep context across AI sessions, and grow toward a scalable company operating system over time.

## Current focus

The current business focus is Izumi IT in the Japanese B2B market.

The first major SaaS/product platform is LINE Business OS.

## Repository structure

- `BOOT.md` - first-read operating context for humans and AI.
- `IAOS_SPECIFICATION.md` - definition of IAOS and what it is not.
- `IAOS_BLUEPRINT.md` - operating areas and repository map.
- `foundation/` - North Star, current state, and transformation strategy.
- `workflows/` - practical repeatable workflows.
- `standards/` - standards that reduce repeated work or risk.
- `decisions/` - architecture and operating decision records.
- `prompts/` - reusable prompts for AI-assisted work.
- `archive/` - retired or historical material.

## How to use this repository

1. Start with `BOOT.md`.
2. Read the relevant foundation, workflow, standard, or decision files.
3. Use the prompt templates when starting AI-assisted work.
4. Update IAOS when a repeated workflow, standard, or decision changes.
5. Keep documents concise and useful for current execution.

## Source of truth

This GitHub repository is the official source of truth for IAOS.

Chat memory, project memory, local notes, and AI conversations may provide helpful context, but they are not official unless the relevant information is captured in this repository.

## Principle

Practical now, scalable later.

IAOS should support real product execution today while steadily growing toward the ideal long-term company operating system.
