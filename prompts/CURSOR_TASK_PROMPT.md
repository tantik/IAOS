# Cursor/Codex Task Prompt

Use this template for implementation tasks.

```text
Goal:
[Describe the exact outcome wanted.]

Context:
[Explain the product, business, user, and repository context.]

Files to inspect:
- [file or directory]
- [file or directory]

Scope:
- [What should change]
- [What should be included]

Non-goals:
- [What should not change]
- [What should not be redesigned]

Constraints:
- Do not invent facts, pricing, legal claims, market data, or technical behavior.
- Preserve existing user work.
- Follow existing repository patterns.
- Keep the change practical and reviewable.
- Do not paste or expose secrets.

Validation:
- [test/lint/typecheck/build/manual check]
- [expected validation result]

Expected output:
- Summary of changed files.
- Diff/stat summary.
- Validation commands and results.
- Assumptions, risks, and follow-up items.
```
