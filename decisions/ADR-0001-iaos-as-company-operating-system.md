# ADR-0001: IAOS as Company Operating System

Status: Accepted
Owner: Konstantin / Izumi IT
Purpose: Record the decision that IAOS is the company operating system for Izumi IT.

## Context

Izumi IT is founder-led and uses AI-assisted development to build products for the Japanese B2B market.

The company needs durable knowledge, workflows, standards, prompts, and decision history that survive individual AI chats, IDE sessions, and tool changes.

## Decision

IAOS is the company operating system for Izumi IT, not just a documentation folder.

It defines how Izumi IT builds products, uses AI, records decisions, manages knowledge, and improves over time.

## Consequences

IAOS documents must remain practical and useful.

The GitHub repository is the source of truth for accepted IAOS knowledge.

AI chats and project memory may help with context, but they are not official unless captured in repository files.

IAOS should grow as Izumi IT grows, without adding abstract structure before it is useful.
